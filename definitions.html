html element definitions

Main Sectioning Elements

HTML : The "html" element represents the root (top-level element) of an HTML document, so it is also referred to as the root element. All other elements must be descendants of this element.

Head : The html "head" element contains machine-readable information (metadata) about the document, lie its title, scripts, and style sheets.

Body : The HTML "body" element represents the content of an HTML document. There can be only one body element in a document.

Document Metadata

Links : The HTML External Resource Link element "link" specifies relationships between the current document and an external resource. This element is most commonly used to link to stylesheets, but is also used to establish site icons (bot "favicon" style icons and icons for the home screen and apps on mobile devices) among other things.

Titles : The HTML Title element "title" defines the document's title that is shown in a browser's title bar or a page's tab.

Content Sections

Footers : The HTML "footer" element represents a footer for its nearest sectioning content or sectioning root element. A footer typically contains information about the author of the section, copyright data or links to related documents.

Headers : The HTML "header" element represents introdctory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.

h1,h2, etc. : The HTML "h1-h6" elements represent six levels of section headings. h1 is the highest section level and h6 is the lowest

Main : The HTML "main" elements represents the dominant content of the body of a document. The main content area consists of content that is directly related to or expands upon the central topic of a document, or the central functionality of an application.

Navigation : The HTML "nav" element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. common examples of navigation sections are menus, tables of contents, and indexes.

Sections : The HTML "section" element represents a standalone section which doesn't have a more specific semantic element to represent it and is contained within an HTML document.

Text Content

Division : The HTML Content Division Element "div" is the generic container for flow content. It has no effect on the content or layout until styled using CSS.

Lists : The HTML "li" element is used to represent an item in a list.

Ordered Lists : The HTML "ol" element represents an orderd list of items, typically rendered as a numbered list.

Paragraphs : The HTML "p" element represents a paragraph. These definitions on this page are inside paragraph elements.

Unordered Lists : The HTML "ul" element represents an unordered list of items, typically rendered as a bulleted list.

Inline Text Semantics

Anchor : The "a" element or anchor element, with it's href attribute, creates a hyperlink to webpages, files, email addresses, locations in the same page, or anything else a URL can address.

Line Break : The HTML "br" element produces a line break in text (carriage-return). It is useful for writing a poem or an address, where the division of lines is significant.

Emphasis : The HTML "em" element marks text that has stress emphasis. The em element can be nested, with each level or nesting indicating a greater degree of emphasis.

Span : The HTML "span" element is a generic inline container for phrasing content, which does not inherently represent anythin. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang.

Strong : The HTML Strong Importance Element "strong" indicates that its contents hve strong importance, seriousness, or urgency. Browsers typically render the contents in bold type.

Images and Multimedia

Images : The HTML "link" element embeds an image into the document.

Table Content

Tables : The HTML "table" element represents tabular date that is, information presented in a two dimensional table comprised of rows and columns of cells containing date.

Table Body : The HTML Table Body element "tbody" encapsulates a set of table rows "tr elements", indicating that they comprise the body of the table.

Table Data Cells : The HTML "td" element defines a cell of a table that contains data. It participates in the table model.

Table Footer : The HTML "tfoot" element defines a set of rows summarizing the columns o the table.

Header Cell Elements : The HTML "th" element defines a cell as header of a group of table cells. The exact nature of this group is defined by the scope and headers attributes.

Table Header : The HTML "thead"

tr Elements : The HTML "tr" element defines a row of cells in a table. The row's of cells can then be established using a mix of td(data cell) and th(header cell) elements

Forms

Buttons : The HTML "button" element represents a clickable button, which can be used in forms or anywhere in a document that needs simple, standard button functionality.

Fieldsets : The HTML "fieldset" element is used to group several contrls as well as labels within the web form.

Forms : The HTML "form" element represents a document section that contains interactive controls for submitting information to a web sever.

Inputs : The HTML "input" element is used to create interactive controls for web based forms in order to accept data from the user, a wide variety of types of input data and control widgets are available, depending on the device and user agent.

Labels : The HTML "label" element represents a caption for an item in a user interface. The title of this section "Labels" is in fact a label.

Legends : The HTML "legend" element represents a caption for the content of its parent fieldset.

Options : The HTML "option" element is used to define an item contained in a select, an optgroup, or a datalis element. As such, option can represent menu items in popups and other lists of items in an HTML document.

Select : The HTML "select" element represents a control that provides a menu of options.

Text Areas : The HTML "textarea" element represents a multi-line plain-text editing control, useful when you want to allow users to enter a sizable amount of free-form tex, for example, a comment on a feview or feeback form.
